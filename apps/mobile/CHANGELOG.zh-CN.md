## [2.0.0-beta.3] - 2026-06-25

### Added

- 通过 iroh mDNS 在同一网络内点对点（p2p）直连 Codux 主机,连接徽章上标明「直连 / 中继」。
- 持久化远程终端:重新回到项目时挂回同一个主机 shell;手机可与桌面共享同一个 agent 终端并交接 viewport(桌面点手机徽章即可收回)。
- 当摄像头实时扫码不顺时,可从相册照片 / 截图导入配对二维码作为兜底。

### Changed

- 扫码配对更清晰:更高摄像头分辨率 + 识别时的加载态;更精简的配对二维码(node id + relay url)并带设备角色徽章。
- 复用暖连接让重连更快,并且不再过早掐断较慢的连接。

### Fixed

- 终端:修复 viewport 尺寸与字形渲染、连接后首行提示符重复 / 残影,以及切换 / 争用时的健壮性;自动恢复卡住的 baseline 以减少延迟导致的卡顿。
- 修复部分 MediaTek/OPLUS 设备的扫码问题(去掉了导致分析饥饿或取景错位的强制摄像头分辨率与自动变焦)。

## [1.8.0] - 2026-06-14

### 新增

- 新增移动端主控链路对共享 Rust protocol FFI 的使用，移动端与桌面端消费同一套协议 payload、transport 元数据、延迟事件和终端布局消息。
- 新增移动端通过 Host runtime model 创建分屏、标签、worktree 和终端布局操作的能力。
- 新增移动端显示设置，应用文字和终端文字分开调整，终端字号提供从小到特大的档位。

### 调整

- 移动端终端屏幕改为从共享 headless terminal model 渲染，不再走独立 Dart 终端历史链路。
- 重构项目、worktree、分屏和标签选择，移动端保留自己的 active UI selection，同时接收共享 Host 的项目 / worktree / 终端关系模型。
- 优化 iOS 和 Android 上的终端滚动、光标绘制、字符宽度测量、输入法转发和 TUI 恢复行为。
- 清理移动端中已由共享 runtime、protocol 和 terminal crates 覆盖的重复兜底逻辑。

### 修复

- 修复切换项目、切换 worktree、后台恢复或桌面 Host 重启后移动端终端偶发空白的问题。
- 修复移动端创建分屏 / 标签不同步到桌面端，以及桌面端创建布局变化不稳定同步到移动端的问题。
- 修复删除分屏 / 标签后移动端残留旧布局状态的问题，并处理删除最后一个终端的无效场景。
- 修复正常实时输出和项目切换时历史加载遮罩反复闪烁的问题。
- 修复大 TUI 会话恢复后只显示一屏、scrollback 丢失或光标 / 输入位置错误的问题。
- 修复输入法退格 / 删除、终端点击聚焦、终端字体回退和移动端光标绘制回归。
- 修复直连 / 中继切换或桌面端重启后延迟和传输路径标签不更新的问题。

## [1.7.5] - 2026-06-09

### 新增

- 新增 v3.1 host capability 解析、终端 buffer 分片组包、按会话维护终端 replica、终端订阅 scope 和大历史有界渲染。
- 新增终端输入可靠发送、输出排序 / 重同步、上传元数据和协议 payload codec，为后续跨设备 runtime domain 做准备。
- 新增移动端更新检查，优化调试日志导出，并补充 Codux Mobile 作为主控端的文档说明。

### 调整

- 将移动端远程工作区拆分为 runtime store、同步、项目、终端、设备、文件、日志、设置和 widget 等独立层，UI 只负责渲染状态和触发用户意图。
- 重构终端页面，当前项目 / session 选择只负责挂载本地 replica，不再决定协议消息是否接收。
- 将项目、文件、worktree、设备操作、调试日志和更新 UI 拆成独立 widget，并补齐对应 service/controller 测试。

### 修复

- 修复首次进入、切换项目、桌面 Host 重启和快速切换分屏时偶发终端空白的问题。
- 修复桌面 Host 重启后移动端可能继续显示旧终端缓存的问题，将 Host 确认的 runtime 状态与 UI 挂载的终端 pane 分离。
- 修复弱网下终端历史恢复不稳定的问题，增加重复分片保护、乱序组包、输入重试和输出重同步辅助逻辑。
- 修复同步回包和列表刷新噪音，重复的 project / terminal list 不再反复触发冲突的 UI 状态变化。

## [1.7.1] - 2026-06-08

## [1.7.0] - 2026-06-08

### 新增

- 正式接入 Codux Mobile v3 远程协议，与 Codux Desktop 共用 Relay / WebRTC 模型。
- 新增无状态二维码 ticket 配对，桌面端通过中继服务发布短期配对载荷。
- 新增 WebRTC DataChannel 直连传输和 WebSocket 中继回退，并通过协议 ping/pong 显示延迟。
- 新增应用内调试日志，支持复制、导出和日志等级配置，方便排查连接与终端显示问题。

### 调整

- 统一项目列表、终端列表、host info、传输状态和重连逻辑，收敛到同一套远程协议状态机。
- 重构首次同步，连接、hello 和传输路径事件都会主动拉取 host info、项目列表、终端列表和终端 buffer。
- 原生终端 controller 创建或 resize 后会回放当前 buffer，避免恢复会话时打开空白终端。

### 修复

- 修复首次配对或重启应用后进入设备，终端为空，必须手动切换项目才显示的问题。
- 修复前后台切换时连接状态跳动、延迟不显示、Relay / P2P 状态标签滞后的问题。
- 修复项目切换和分屏选择时重复请求终端 buffer 与回放顺序异常的问题。

### 说明

- Codux Mobile 1.7.0 建议搭配 Codux Desktop 1.7.0 使用，升级后建议重新配对已有设备。
