# Codex 宠物 Atlas 规范

Codux 宠物统一使用 Codex 风格 atlas。最终导入到项目里的图片必须是固定坐标雪碧图：`8 列 x 9 行`，总尺寸 `1536 x 1872 px`，单帧格子 `192 x 208 px`。

## 设计原则

新增宠物只做“单形态、多动作”：

- 不需要蛋形态。
- 不需要幼年、成年、进化、终极形态。
- 不需要单独睡眠阶段。
- 只需要同一个角色的一张完整动作 atlas。
- 领取后直接显示宠物，后续等级、经验、图鉴、提示气泡等系统能力可以继续使用。

这样新增宠物的成本最低，也最适合用图片模型生产。多形态很容易导致服装、比例、颜色、五官不一致，所以不再作为默认素材规范。

已有内置宠物的物种、等级、经验、图鉴和历史数据需要保留，但素材也统一改成单形态 atlas。也就是说，之前的宠物仍然是原来的物种和进度，只是每个物种换成一张新的单形态图片，不再按蛋、幼年、成年、进化分别生产素材。

## 文件结构

外部自定义宠物放在：

```text
${CODEX_HOME:-$HOME/.codex}/pets/<pet-name>/
├── pet.json
└── spritesheet.png
```

`pet.json` 示例：

```json
{
  "id": "hoodie-cat",
  "displayName": "连帽猫",
  "description": "一只陪你写代码的黑白连帽猫。",
  "spritesheetPath": "spritesheet.png"
}
```

内置物种也按同一套结构维护，直接放在 `Sources/DmuxWorkspace/Resources/Pets/<species>/`。运行时只读取物种级 flat package，不再读取阶段目录。

## Atlas 规格

- 总画布：`1536 x 1872 px`
- 网格：`8 列 x 9 行`
- 单帧格子：`192 x 208 px`
- 背景：生成图使用纯白背景，导入项目前用脚本转换成真实透明背景
- 不要网格线、不要棋盘格、不要行列标签、不要文字、不要水印
- 不要投影、接触阴影、外发光、内发光、半透明烟雾、半透明粒子、模糊边缘、复杂地面或反射

图片模型统一生成纯白或近白背景，再用仓库脚本转换成透明背景。图片模型不一定必须一次输出 `1536 x 1872 px`，但必须保持清楚的 `8 列 x 9 行` 排布，不能斜切、错行、合并格子或让角色跨格。

## 动作行

| 行号 | 动作 | 帧数 | 说明 |
| --- | --- | --- | --- |
| 0 | `idle` | 6 | 待机、轻微呼吸、尾巴或耳朵轻动 |
| 1 | `running-right` | 8 | 向右跑步 |
| 2 | `running-left` | 8 | 向左跑步 |
| 3 | `waving` | 4 | 打招呼 |
| 4 | `jumping` | 5 | 下蹲、起跳、腾空、落地、回稳 |
| 5 | `failed` | 8 | 失败、沮丧、生气、无害受挫 |
| 6 | `waiting` | 6 | 等待、思考、困倦或睡眠 |
| 7 | `running` | 6 | 原地忙碌、处理中 |
| 8 | `review` | 6 | 查看、阅读、检查、确认 |

未使用的格子必须保持透明：

- 第 0 行只用第 0-5 列，第 6-7 列透明。
- 第 3 行只用第 0-3 列，第 4-7 列透明。
- 第 4 行只用第 0-4 列，第 5-7 列透明。
- 第 6、7、8 行只用第 0-5 列，第 6-7 列透明。

睡眠不需要额外图片。Codux 会把睡眠渲染映射到第 6 行 `waiting`，所以第 6 行可以画成等待、思考、困倦或轻微打盹。

## 新宠物提示词

把下面整段发给图片模型。需要替换尖括号里的内容。

```text
请生成一张用于编程助手应用的像素风电子宠物雪碧图。

核心要求：
- 这只宠物只有一种形态。
- 不要蛋形态。
- 不要幼年、成年、进化、终极形态。
- 不要变身，不要多个角色。
- 全图都必须是同一个角色，只是动作不同。

重要输出要求：
- 只输出一张完整 PNG 图片。
- 必须使用纯白背景，不要透明背景，不要棋盘格背景，不要渐变背景。
- 总画布尺寸优先使用 1536 x 1872 像素；如果做不到，也必须保持严格的 8 列 x 9 行。
- 画布必须严格切成 8 列 x 9 行。
- 如果总画布是 1536 x 1872 像素，每个格子尺寸必须是 192 x 208 像素。
- 每一帧都只能有同一个角色。
- 不要文字、不要水印、不要 UI、不要边框、不要场景背景、不要行列标签、不要可见网格线。
- 不要气泡、对话框、表情气泡、图标、星星、汗滴、音符、问号、感叹号、速度线、魔法阵、粒子、漂浮小物件或任何宠物周围装饰。
- 不要投影、接触阴影、外发光、内发光、半透明烟雾、半透明粒子、模糊边缘、复杂地面或反射。
- 角色边缘必须干净利落，适合从纯白背景中自动抠图。

宠物设定：
- 宠物名称：<宠物名称>
- 物种/主题：<猫、狗、鸟、机器人、精灵、史莱姆等>
- 性格：<安静、调皮、勇敢、害羞、困倦、专注等>
- 主色：<2 到 4 个颜色>
- 标志特征：<耳朵、尾巴、连帽衫、护目镜、小电脑、徽章、围巾等>，类似宠物小精灵，不要太像真实动物，要拟人化；除了奔跑等方向性动作，其他空闲类动作至少要正面向前

画风要求：
- 精致像素风。
- 可爱的桌面陪伴型宠物。
- 轮廓清晰，缩小后也能看清。
- 每一帧角色外观必须一致。
- 服装、颜色、五官、比例、尾巴、配饰在所有帧中保持一致。
- 每个格子四周保留少量透明留白。
- 同一行动画的脚底基线和身体中心点必须稳定，不要让角色在格子里随机漂移。
- 除了跳跃动作必需的轻微垂直变化，不要让角色在帧之间产生明显位移。
- 同一行动作必须连贯，上一帧到下一帧只能是小幅自然变化，不要突然换姿势、换角度、换比例或换表情幅度过大。
- 每一行动作都要能循环播放，第一帧和最后一帧的位置、姿态、表情要自然衔接，不要出现首尾断裂。
- 角色不要被裁切。
- 不要画软阴影或发光描边；如果需要强调轮廓，只能使用角色自身的实色像素描边。

动作行要求：
- 第 0 行：待机 idle，使用 6 帧，第 6-7 列留空透明。动作是轻微呼吸、耳朵或尾巴轻轻摆动。
- 第 1 行：向右跑 running-right，使用 8 帧。动作是明确朝右的跑步循环。
- 第 2 行：向左跑 running-left，使用 8 帧。动作是明确朝左的跑步循环，和第 1 行保持镜像感。
- 第 3 行：打招呼 waving，使用 4 帧，第 4-7 列留空透明。动作是友好挥手或挥爪。
- 第 4 行：跳跃 jumping，使用 5 帧，第 5-7 列留空透明。动作是下蹲、起跳、腾空、落地、回稳。
- 第 5 行：失败 failed，使用 8 帧。动作是无害的沮丧、生气、受挫或失败反应，不要血腥，不要受伤细节。
- 第 6 行：等待 waiting，使用 6 帧，第 6-7 列留空透明。动作是等待、思考、困倦或睡眠，可带轻微打盹效果。
- 第 7 行：忙碌 running，使用 6 帧，第 6-7 列留空透明。动作是原地忙碌、快速处理、准备冲刺。
- 第 8 行：检查 review，使用 6 帧，第 6-7 列留空透明。动作是查看、阅读、检查、确认，可加入小电脑或文档动作。

质量约束：
- 不要让角色在帧之间随机漂移。
- 不要让同一行动作的第一帧和最后一帧无法连起来。
- 不要让同一动作里出现突然变大、变小、变高、变矮、换脸、换服装或换视角的大差别。
- 不要在角色周围添加任何装饰性元素，包括气泡、符号、星星、汗滴、速度线、魔法特效和漂浮道具。
- 不要在不同行之间改变服装、颜色、物种特征。
- 不要把多个格子合并成一张大图。
- 不要让任意一帧越界或被裁切。
- 最终图片必须看起来像一个 8 列 x 9 行的动作雪碧图，但不要显示网格线。
```

## 白底图片转换

生成结果应为纯白背景，使用这个脚本把它标准化成 Codux/Codex 可用包。

脚本依赖 Pillow。如果本机没有安装，先执行：

```bash
python3 -m pip install Pillow
```

转换命令：

```bash
python3 scripts/pet-sprites/normalize_codex_atlas.py \
  ~/Downloads/hoodie-cat-white.png \
  --output-dir "$HOME/.codex/pets/hoodie-cat" \
  --id hoodie-cat \
  --name 连帽猫 \
  --description "一只陪你写代码的黑白连帽猫。"
```

替换内置物种素材时，把输出目录换成这个物种的单形态素材目录，例如：

```bash
python3 scripts/pet-sprites/normalize_codex_atlas.py \
  ~/Downloads/voidcat-white.png \
  --output-dir "Sources/DmuxWorkspace/Resources/Pets/voidcat" \
  --id voidcat \
  --name "喵喵" \
  --description "Codux bundled pet atlas."
```

脚本会做这些事：

- 按 `8 列 x 9 行` 切分原图。
- 对每个格子去除纯白或近白背景。
- 把每帧重新缩放、居中到底部稳定的 `192 x 208 px` 单元格。
- 清空未使用的尾部格子。
- 输出标准 `spritesheet.png` 和 `pet.json`。

如果白底没有完全去干净，可以调大容差：

```bash
python3 scripts/pet-sprites/normalize_codex_atlas.py \
  ~/Downloads/hoodie-cat-white.png \
  --output-dir "$HOME/.codex/pets/hoodie-cat" \
  --id hoodie-cat \
  --name 连帽猫 \
  --tolerance 36
```

如果角色被缩得太小或太大，可以调整内容比例：

```bash
python3 scripts/pet-sprites/normalize_codex_atlas.py \
  ~/Downloads/hoodie-cat-white.png \
  --output-dir "$HOME/.codex/pets/hoodie-cat" \
  --id hoodie-cat \
  --name 连帽猫 \
  --content-scale 0.86
```

## 验收标准

拿到图以后先检查：

- 是否正好 8 列 9 行。
- 是否只有同一个角色、同一个年龄、同一套衣服、同一比例。
- 是否没有蛋、幼年、成年、进化、变身或其他角色。
- 如果尺寸不是 `1536 x 1872 px`，行列是否仍然整齐，能否被脚本等分裁切。
- 背景是否是纯白/近白，而不是透明棋盘格、渐变或复杂背景。
- 是否没有投影、接触阴影、外发光、半透明烟雾、半透明粒子、模糊边缘、复杂地面或反射。
- 角色边缘是否干净，去白底后不会留下灰边或白边。
- 第 0、3、4、6、7、8 行的末尾空格是否透明或纯白空白。
- 第 6 行是否能作为等待或睡眠动作。
- 第 8 行是否能作为 review/检查动作。

满足这些条件，就可以转换成 Codux/Codex 兼容宠物 atlas 使用。

如果是替换已有内置物种素材，还要额外检查：

- 物种标识不要改，例如 `voidcat` 仍然是 `voidcat`。
- 不要把原物种拆成多个阶段素材。
- 不要删除或重置已有宠物状态数据。
- 旧等级和进度只影响系统能力，不再要求改变宠物外观。
